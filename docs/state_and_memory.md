# State & Memory Management

## `AgentState`

`AgentState` is the serializable snapshot of everything that happened during an agent's lifetime. It lives on the `StatefulAgent` and is updated after every turn.

Key fields:

| Field | Type | Description |
|-------|------|-------------|
| `sessionId` | `String` | Unique identifier for this session |
| `history` | `AgentMessageHistory` | Conversation messages + episodic memories |
| `usages` | `List<ModelUsage>` | Token usage records for all LLM calls |
| `metadata` | `Map<String, dynamic>` | User-defined data (e.g. user ID, preferences) |
| `plan` | `PlanState?` | Current todo list from the Planner |
| `activeSkills` | `List<String>?` | Names of currently active skills |
| `isRunning` | `bool` | Whether the agent is mid-run (used for resume) |
| `totalLoopCount` | `int` | Total LLM calls across all runs |
| `currentLoopCount` | `int` | LLM calls in the current run |
| `currentLoopUsages` | `List<ModelUsage>` | Token usage records for the current run only |
| `lastError` | `String?` | Error message from the last failed run |
| `systemReminders` | `Map<String, String>` | Dynamic per-request notes injected before the last user message |
| `systemPromptHistory` | `List<SystemPromptHistoryItem>` | Snapshot of system prompt per message index |
| `toolsHistory` | `List<ToolsHistoryItem>` | Snapshot of tool list per message index |

> Note: The `Tool` and `Skill` *definitions* are held by `StatefulAgent`, not `AgentState`. `AgentState.activeSkills` only stores the names of currently active skills as strings.

## `AgentCallToolContext`

When the agent executes a tool, it wraps the call in a Dart `Zone` carrying an `AgentCallToolContext`. This allows tool functions to access session state without needing explicit parameters:

```dart
String checkBalance(String currency) {
  final context = AgentCallToolContext.current;
  final userId = context?.state.metadata['user_id'] as String?;
  return fetchBalance(userId, currency);
}
```

Available via `AgentCallToolContext.current`:
- `state`: The live `AgentState`
- `agent`: The `StatefulAgent` running the tool
- `batchCallId`: Shared UUID for all tools in one parallel batch
- `cancelToken`: The current run's `CancelToken`

## `FileStateStorage`

`FileStateStorage` persists `AgentState` as a JSON file on disk. Each session maps to one file named `$sessionId.json` inside the given directory.

```dart
final storage = FileStateStorage(Directory('.state_dir'));

// Load existing state, or create a new one with initial metadata
final state = await storage.loadOrCreate('session_123', {
  'user_id': 'u_001',
  'premium_status': 'gold',
});

final agent = StatefulAgent(
  name: 'my_agent',
  client: client,
  modelConfig: modelConfig,
  state: state,
  // Called after each tool batch completes and in the finally block
  autoSaveStateFunc: (s) async => await storage.save(s),
);
```

Other `FileStateStorage` methods:

```dart
await storage.save(state);             // explicit save
await storage.delete('session_123');   // delete a session
final exists = await storage.exist('session_123');
```

### Metadata Merging

When loading an existing session with `loadOrCreate`, any `initialMetadata` you pass is **merged into** the existing metadata by default (existing keys are overwritten). Pass `overwrite: false` to skip merging if the session already exists.

## `LLMBasedContextCompressor`

For long-running sessions, the active context window will eventually exceed the model's token limit. Attach a `LLMBasedContextCompressor` to handle this automatically.

When the token count (measured from the last `ModelUsage.promptTokens`) exceeds `totalTokenThreshold`, the compressor:
1. Takes all but the most recent `keepRecentMessageSize` messages
2. Sends them to the LLM with a summarization prompt
3. Stores the result as an `EpisodicMemory` (a compressed XML snapshot + the original raw messages)
4. Replaces the old messages with a short summary placeholder in the active context

```dart
final compressor = LLMBasedContextCompressor(
  client: client,
  modelConfig: ModelConfig(model: 'gpt-4o-mini'),
  totalTokenThreshold: 64000, // default
  keepRecentMessageSize: 10,  // default
);

final agent = StatefulAgent(..., compressor: compressor);
```

The compressor always keeps `FunctionCall`/`FunctionExecutionResult` pairs intact — it never splits a tool call from its result when deciding what to compress.

---

## Episodic Memory

Compressed history is stored as `EpisodicMemory` entries inside `AgentState.history.episodicMemories`. Each entry has:
- `id`: A short random ID (e.g. `episode_aB3xYz`)
- `summary`: The XML state snapshot generated by the compressor
- `messages`: The original raw messages before compression

The summary is injected back into the active context as a `UserMessage`, so the model always has a structural overview of past events. The model also sees the episode ID and is told it can call `retrieve_memory` to access the full original messages if needed.

### `retrieve_memory` Tool

When episodic memories exist, the agent automatically gains access to the `retrieve_memory` tool:

```dart
// The agent calls this internally when it needs precise historical details
// Tool parameters:
// - snapshot_id (required): e.g. 'episode_aB3xYz'
// - limit (optional): max messages to return (default 20)
// - offset (optional): pagination offset (default 0)
```

This lets the agent retrieve verbatim past messages for cases where the summary isn't detailed enough, without reloading everything into the active context.

## `systemReminders`

`AgentState.systemReminders` is a `Map<String, String>` of short notes injected into the request before each LLM call. The reminders are wrapped in `<system-reminders>` XML tags and inserted as a `UserMessage` just before the last `UserMessage` in the request message list.

```dart
agent.state.systemReminders['current_time'] = DateTime.now().toIso8601String();
agent.state.systemReminders['rate_limit'] = 'User has 3 requests remaining today.';
```

Reminders persist in state and are included in every subsequent LLM call until removed. To remove a reminder, delete its key from the map.
